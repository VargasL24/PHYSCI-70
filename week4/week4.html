Week 4: Microcontroller Programing

For this week my first idea was to build onto my week 3 project, and make a record player that would be able to slow down and spin for a given time to simmulate a song. However due to all of the internals being unaccessable and the motor that was mounted breaking this was no longer and option for me. What I came up with was a mini trashcan that could be opened and closed with the push of a button.

For my design I wanted it to be as compact as possible but to also make sure that the trash doesnt touch any of the electromics in the can itself. I also knew that I would want this to be opened easily to acccess the electronics incase something malfunctions. In my design I ended up going for a revolving lid idea that would have a gap to avoid frition slowing down or stoping the lid.

First I laser cut the parts of the trash can itself, I designed everything in fusion 360 because it is quick and easy, and would give me the prrecise angles I needed to make the box. The container that would hold all the trash was a plastic cup so that trash that was being tossed wouldnt damage anything inside.

For the internals what I did was mount the motor to the side of the box using hot glue and used a stick to connect the motor to the lid. using the arduino and a breadboard I connected the motor to port 3 and 4 and connected the other two wires to the ground and 5V ports.There was a button that when pressed gave the lid a half turn allowing the user to throw away trash. This is the code used to tell the arduino what to do.


const int A1A = 3;  // define pin 3 for A-1A 
const int A1B = 4;  // define pin 4 for A-1B 

void setup() {
  pinMode(A1A, OUTPUT);     // specify these pins as outputs
  pinMode(A1B, OUTPUT);
  digitalWrite(A1A, LOW);   // start with the motors off 
  digitalWrite(A1B, LOW);
}

void loop() {
  // start the motor 
  digitalWrite(A1A, HIGH);   
  digitalWrite(A1B, LOW);
  delay(75);              // allow the motor to run for 4 seconds

  // stop the motor
  digitalWrite(A1A, LOW);   // setting both pins LOW stops the motor 
  digitalWrite(A1B, LOW);   // redundant, but doesn't hurt 
  delay(2500);              // keep the motor off for 2 seconds
}

What I learned

I learned the different ways to connect ardino to the motors and now to use buttons to communicate to the motor. It was fun using what I learned from the previous project to improve apon this weeks assignment.
